# Preventive
1. Stabilitas dan Ketersediaan Sistem Selama Masa Kritis
2. Alasan: Mencegah sistem down, crash, atau lambat saat diakses secara massal oleh ribuan mahasiswa dalam waktu bersamaan.
3. Aksi Nyata: Melakukan uji beban sebelum masa KRS/UTM, mengaktifkan cadangan server (load balancing), dan menyiapkan skema pembagian jadwal akses per angkatan/jurusan.

# Corrective
1. Validasi Akurasi dan Keamanan Data Nilai 
2. Alasan: Menjamin tidak ada kesalahan hitung pada IPK/SKS dan mencegah manipulasi data akademis yang bersifat sensitif.
3. Aksi Nyata: Memperbaiki pembulatan rumus nilai secara berkala, melakukan audit log, serta membatasi hak akses pengisian nilai hanya untuk dosen pengampu dan kaprodi.

# Adaptive
1. Kepatuhan Regulasi dan Integrasi Eksternal
2. Alasan: Memastikan data kampus selalu sinkron dengan sistem pemerintah pusat dan memudahkan transaksi keuangan mahasiswa.
3. Aksi Nyata: Memperbarui modul ekspor data agar selalu sesuai dengan format terbaru Kemendikbud serta memperbarui API pembayaran (Host-to-Host) dengan bank-bank mitra.

# perfective
1. Peningkatan Efisiensi Antarmuka dan Kecepatan Akses
2. Alasan: Memastikan sistem tidak hanya sekadar "berjalan", tetapi juga nyaman, cepat, dan mudah digunakan oleh mahasiswa maupun dosen, sehingga mengurangi kebingungan dan beban kerja bagian helpdesk.
3. Menambahkan fitur notifikasi otomatis melalui WhatsApp atau Email ketika dosen selesai menginput nilai, sehingga mahasiswa tidak perlu mengecek sistem secara berulang-ulang.
